# Установка Arch Linux 

> [!NOTE]
> Этот гайд __НЕ__ затрагивает все возможные варианты установки `Arch Linux`. Используя это руковдство вы получите ___голый___ `Arch Linux` без окружения рабочего стола или композитора. В тексте будут встречаться примечания, как правило они будут содержать дополнительные пояснения или ___краткое___ описание других возможных вариантов установки c использованием других инструментов.   

> [!WARNING]
> Гайд предполагает что `Arch Linux` будет единственной системой на машине. Он __НЕ__ описывает `Dual boot` c `Windows`.  Все данные на диске будут уничтожены.

__Получаемый стэк:__
* __Прошивка__: только `UEFI`
* __Загрузчик__: `systemd-boot` 
* __Сеть__:
  * Разрешение DNS: `systemd-resolved`
  * WiFi: `iwd` (используется встроенный `DHCP-клиент`)
  * Ethernet: нет 
* __Структура разделов__:
  ```
  /dev/nvme0n1
  ├── /dev/nvme0n1p1 → /boot (FAT32, 1 ГБ)
  ├── /dev/nvme0n1p2 → / (ext4, min 30 ГБ)
  ├── /dev/nvme0n1p3 → swap (равен объёму RAM, гибернация)
  └── /dev/nvme0n1p4 → /home (ext4, всё оставшееся место)
  ```
> [!NOTE]
> 1. Если у вас машина с  `Ethernet-разьемом` то используйете или `Network Manager` или связку `systemd-netword` + `systemd-resolved` + `iwd`(или `wpa supplikant`)
> 2. `GRUB` вместо `systemd-boot`
